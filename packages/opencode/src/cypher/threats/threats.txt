## Instructions

Analyze the repository to construct a threat model using standard methodologies (like STRIDE). Focus on:
1. **Trust Boundaries & Data Flows:** Identify entry points (API endpoints, public ports, forms, sockets) and where data crosses from untrusted to trusted environments.
2. **Authentication & Access Control:** Assess how sessions are handled, passwords/tokens are validated, and permissions are enforced.
3. **Data Storage & Injection Risks:** Review how the application queries databases, reads/writes local files, or processes external input (to spot SQLi, NoSQLi, Command Injection, SSRF, XSS, or Path Traversal risks).
4. **Secrets Leakage:** Scan for hardcoded API keys, private keys, passwords, or sensitive local files exposed in the repository.
5. **Supply Chain Risks:** Review imports and package configuration files (package.json, requirements.txt, go.mod, Cargo.toml) for insecure dependencies or obsolete libraries.
